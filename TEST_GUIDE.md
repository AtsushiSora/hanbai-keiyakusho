# 販売契約書 テスト説明書

販売契約書システムを公開前・公開後に確認するための手順です。

## 重要な前提

このシステムは、販売契約の情報を入力フォームに入れて、写真ベースの帳票型PDFへ転記し、PDF保存または印刷する流れです。

- GitHub Pagesで確認できること: 画面表示、テスト用ログイン、入力、帳票型PDFへの転記、PDF印刷画面
- Netlifyで確認すること: 管理者ログイン、クラウド保存、契約一覧、削除

GitHub PagesではNetlify Identity、Netlify Functions、Netlify Blobsが動かないため、クラウド保存は確認できません。

## 1. 画面確認

確認URL:

```text
https://atsushisora.github.io/hanbai-keiyakusho/
```

チェック項目:

- 管理者ログイン画面が表示される
- `テスト用ログイン` で作成画面に入れる
- `EDITOR / 新規契約作成` の操作パネルが表示される
- スマホ幅でも文字やボタンが重ならない

## 2. 入力確認

テスト用ログイン後、`契約書作成` を開いて以下を入力します。

買主情報:

```text
氏名: 山田 太郎
電話番号: 090-1111-2222
メールアドレス: test@example.com
住所: 広島県広島市中区テスト1-2-3
```

車両情報:

```text
車名: スズキ ワゴンR
グレード: FX
年式: 2018年
型式: MH55S
車台番号: MH55S-123456
登録番号: 広島580あ1234
走行距離: 82,000km
車検満了日: 令和8年5月
車体色: 白
修復歴: なし
```

金額:

```text
車両本体価格: 300000
諸費用: 50000
税金・保険料等: 20000
リサイクル預託金: 9000
値引き: 10000
```

チェック項目:

- 入力欄に問題なく入力できる
- `下書き保存` で端末内履歴に保存できる
- `入力クリア` で新規入力に戻せる

## 3. 帳票型PDF確認

操作:

1. 入力フォームにテスト情報を入れる
2. `PDF保存` を押す
3. 帳票型PDFページへ移動する
4. 入力内容が帳票上に転記されているか確認する
5. 帳票ページの `印刷・PDF保存` を押す

チェック項目:

- 買主名、住所、電話番号が転記される
- 車名、グレード、年式、型式、車台番号が転記される
- 金額が転記される
- 文字が枠から大きくズレていない
- 印刷対象が帳票1枚になる

## 4. PDF印刷確認

操作:

1. 入力フォームにテスト情報を入れる
2. `PDF印刷` を押す
3. 帳票型PDFページへ移動後、印刷画面を確認する

チェック項目:

- 旧契約書プレビューではなく、写真ベースの帳票が印刷対象になる
- 用紙サイズはA4
- 余白や縮尺で帳票が切れていない

## 5. クラウド保存確認

Netlify公開環境で確認します。

操作:

1. 管理者ログインする
2. 契約情報を入力する
3. `クラウド保存` を押す
4. `契約一覧` から保存済み契約を確認する

チェック項目:

- 保存成功メッセージが出る
- 保存済み契約一覧に表示される
- 一覧から読み込める
- 削除できる

## 6. 本番前の確認

- テスト用ログインを無効化する
- 管理者ログインで入れることを確認する
- 管理者以外が契約画面を見られないことを確認する
- 帳票PDFの入力位置を実車両データで確認する

テスト用ログインを無効化する場合:

```js
const ENABLE_TEST_LOGIN = false;
```

変更後は `npm run build` を実行して `contract-auth.bundle.js` を更新します。
