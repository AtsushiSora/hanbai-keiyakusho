# 販売契約書 テスト説明書

販売契約書システムを公開前・公開後に確認するための手順です。

## 重要な前提

このシステムはGitHub Pagesで画面を公開し、ログインと契約データ保存にSupabaseを使います。Netlifyは利用しません。

- 確認できること: 管理者ログイン、入力、クラウド保存、契約一覧、JSON出力・取込、PDF保存・印刷、メール・LINE文面作成
- 保存場所: Supabaseの契約・リモート契約・監査イベント用テーブル
- 注意点: `src/supabase-config.js` のProject URLとPublishable key設定、`supabase-schema.sql` の実行が必要です。

## 0. Supabase準備

1. Supabaseでプロジェクトを作成する
2. SQL Editorで `supabase-schema.sql` を実行する
3. Authenticationで管理者ユーザーを作成する
4. Project URL と Publishable key を `src/supabase-config.js` に設定する
5. `enableTestLogin` を本番では `false` にする
6. GitHubへ反映してGitHub Pagesで読み込める状態にする
7. SupabaseのSite URLを `https://atsushisora.github.io/hanbai-keiyakusho/admin-invite.html` に設定する

チェック項目:

- `order_auto_contracts` テーブルがある
- `order_auto_remote_contracts` と `order_auto_remote_events` テーブルがある
- RLSが有効になっている
- 管理者ユーザーでログインできる
- Secret key、`service_role` キーを公開ファイルに入れていない

## 1. 画面確認

確認URL:

```text
https://atsushisora.github.io/hanbai-keiyakusho/
```

チェック項目:

- 管理者ログイン画面が表示される
- Supabaseに登録したメールアドレスとパスワードで入れる
- テスト期間中だけ `テスト用ログイン` で作成画面に入れる
- `EDITOR / 新規契約作成` の操作パネルが表示される
- スマホ幅でも文字やボタンが重ならない

## 2. 入力確認

ログイン後、`契約書作成` を開いて以下を入力します。

買主情報:

```text
氏名: 山田 太郎
電話番号: 090-1111-2222
メールアドレス: test@example.com
住所: 広島県広島市中区テスト1-2-3
```

車両情報:

```text
車名: スズキ ワゴンR
グレード: FX
年式: 2018年
型式: MH55S
車台番号: MH55S-123456
登録番号: 広島580あ1234
走行距離: 82,000km
車検満了日: 令和8年5月
車体色: 白
修復歴: なし
```

金額:

```text
車両本体価格: 300000
諸費用: 50000
税金・保険料等: 20000
リサイクル預託金: 9000
値引き: 10000
```

チェック項目:

- 入力欄に問題なく入力できる
- `下書き保存` でブラウザ内の一時下書きに保存できる
- `クラウド保存` でSupabaseに保存できる
- `入力クリア` で新規入力に戻せる

## 3. 契約一覧確認

操作:

1. `契約一覧` を開く
2. 保存済み契約が表示されることを確認する
3. `編集` を押して作成ページへ戻る
4. `メール・LINE契約` を押して送信準備ページへ移動する
5. `削除` で不要な契約を削除できるか確認する

チェック項目:

- 検索できる
- `すべて / 下書き / 署名待ち / 送信済み / 完了` の絞り込みが動く
- JSON出力・JSON取込ができる

## 4. PDF保存確認

操作:

1. 入力フォームにテスト情報を入れる
2. `PDF保存` を押す
3. PDFテンプレートページへ移動する
4. 入力内容が帳票上に転記されているか確認する
5. 帳票ページの `印刷・PDF保存` を押す

チェック項目:

- 買主名、住所、電話番号が転記される
- 車名、グレード、年式、型式、車台番号が転記される
- 金額が転記される
- 文字が枠から大きくズレていない
- 印刷対象が帳票1枚になる

## 5. PDF印刷確認

操作:

1. 入力フォームにテスト情報を入れる
2. `PDF印刷` を押す
3. PDFテンプレートページへ移動後、印刷画面を確認する

チェック項目:

- 写真ベースの帳票が印刷対象になる
- 用紙サイズはA4
- 余白や縮尺で帳票が切れていない

## 6. メール・LINE契約確認

操作:

1. 契約作成ページで入力して `クラウド保存` を押す
2. `契約一覧` から `メール・LINE契約` を押す
3. `確認URL生成` を押す
4. お客様確認URLと開封パスコードが表示される
5. `URLコピー`、`パスワードコピー`、`LINE文面コピー` が使えるか確認する
6. `メール作成` でメール本文が作られるか確認する
7. 生成された確認URLを開き、別送用の開封パスコードで契約内容を開けるか確認する

チェック項目:

- 契約一覧で選んだ契約内容が引き継がれる
- URLとパスコードは別々に送る案内になっている
- お客様確認ページで契約概要、重要事項、電子署名が表示される
- 電子署名完了後、`order_auto_remote_contracts` に署名者名・同意内容・署名画像・完了日時が保存される
- 元の契約が `完了` 状態になる
- 誤ったパスコードを5回入力すると15分間ロックされる

## 7. 本番前の確認

- `src/supabase-config.js` の `enableTestLogin` を `false` にする
- Supabaseの管理者ユーザーを強いパスワードにする
- SupabaseのRLSポリシーが有効なことを確認する
- GitHubにSecret key、`service_role` キー、管理者パスワードが含まれていないことを確認する
- 契約一覧からJSON出力してバックアップを取る
- 帳票PDFの入力位置を実車両データで確認する
