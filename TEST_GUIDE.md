# 販売契約書 テスト説明書

この説明書は、販売契約書システムを公開前・公開後に確認するための手順です。

## 重要な前提

このシステムには2種類の確認があります。

- GitHub Pagesで確認できること: 画面表示、入力欄、契約書プレビュー、印刷レイアウト
- Netlifyで確認すること: 管理者ログイン、契約一覧、サーバー保存、サーバー削除

GitHub PagesではNetlify Identity、Netlify Functions、Netlify Blobsが動かないため、ログイン後の保存機能は確認できません。

## 1. GitHub Pagesで画面を確認

確認URL:

```text
https://atsushisora.github.io/hanbai-keiyakusho/
```

チェック項目:

- ページが表示される
- ロゴや背景画像が表示される
- 管理者ログイン画面が表示される
- 画面崩れがない
- スマホ幅でも文字やボタンが重ならない

注意:

GitHub Pagesでは本番ログイン処理は動きません。画面確認には `テスト用ログイン` ボタンを使ってください。正式なログイン機能の確認はNetlifyで行います。

## 1-1. テスト用ログインで確認

操作:

1. `テスト用ログイン` を押す
2. 契約入力フォームが表示される
3. 入力、印刷、PDF保存、タブレット署名を確認する

チェック項目:

- ログイン登録なしで入力画面が表示される
- 管理者メニューに `テスト用ログイン中` と表示される
- サーバー保存を押すと、テスト中は保存不可の案内が出る
- ログアウトするとログイン画面に戻る

本番公開前の対応:

`src/contract-auth.js` の下記を `false` に変更し、`npm run build` を実行してください。

```js
const ENABLE_TEST_LOGIN = false;
```

## 2. Netlifyで公開設定を確認

Netlifyで新規サイトを作成し、このGitHubリポジトリを接続します。

設定:

```text
Build command: npm run build
Publish directory: .
```

Netlifyの環境変数:

```text
ADMIN_EMAILS=管理者のメールアドレス
```

複数の管理者を入れる場合:

```text
ADMIN_EMAILS=admin1@example.com,admin2@example.com
```

Identity設定:

- Identityを有効化
- Registrationは Invite only 推奨
- 管理者ユーザーを招待
- 管理者ユーザーに admin ロールを付与、または `ADMIN_EMAILS` にメールを登録

## 3. 管理者ログインの確認

Netlifyの公開URLで `contract.html` を開きます。

チェック項目:

- 管理者ログイン画面が表示される
- 登録したメールアドレスとパスワードでログインできる
- ログイン後に契約入力フォームが表示される
- ログアウトできる
- ログアウト後に入力画面が見えなくなる

失敗した場合:

- Identityが有効になっているか確認
- 招待メールの認証が完了しているか確認
- `ADMIN_EMAILS` のメールアドレスが一致しているか確認
- adminロールが付いているか確認

## 4. 契約書入力の確認

以下のテスト情報を入力します。

買主情報:

```text
氏名: 山田 太郎
電話番号: 090-1111-2222
メールアドレス: test@example.com
住所: 広島県広島市中区テスト1-2-3
```

車両情報:

```text
車名: スズキ ワゴンR
グレード: FX
年式: 2018年
型式: MH55S
車台番号: MH55S-123456
登録番号: 広島580あ1234
走行距離: 82,000km
車検満了日: 令和8年5月
車体色: 白
修復歴: なし
```

金額:

```text
車両本体価格: 300000
諸費用: 50000
税金・保険料等: 20000
リサイクル預託金: 9000
値引き: 10000
```

チェック項目:

- 入力内容が契約書プレビューへ反映される
- 総支払額が自動計算される
- 契約日が表示される
- 売主情報がオーダーオートになっている
- 電話番号が `080-2912-8616` になっている

## 5. 印刷・PDF保存の確認

操作:

1. `印刷・PDF保存` を押す
2. 印刷画面でPDF保存を選ぶ
3. 保存したPDFを開く

チェック項目:

- 契約書部分だけが印刷される
- 入力フォームやボタンが印刷されない
- 罫線が表示される
- 署名欄が表示される
- 文字が重なっていない

## 6. タブレット署名の確認

操作:

1. 完結方法で `タブレット署名` を選ぶ
2. 署名欄に指またはマウスでサインを書く
3. 契約書プレビューを確認する

チェック項目:

- 署名欄が表示される
- サインが書ける
- サインが契約書プレビューに反映される
- `消去` ボタンでサインを消せる
- サイン入りでPDF保存できる

## 7. メール完結の確認

操作:

1. 完結方法で `メール完結` を選ぶ
2. `メール下書き` を押す

チェック項目:

- メールアプリが開く
- 宛先に買主メールアドレスが入る
- 件名に契約確認の内容が入る
- 本文に契約概要が入る

注意:

PDF添付は自動では行いません。PDF保存後、メールに添付してください。

## 8. サーバー保存の確認

Netlify公開環境で確認します。

操作:

1. 管理者ログインする
2. 契約情報を入力する
3. `サーバーに保存` を押す
4. `契約一覧を更新` を押す

チェック項目:

- 保存成功メッセージが出る
- 保存済み契約一覧に契約が表示される
- 一覧から契約を選んで読み込める
- 読み込んだ内容がフォームに復元される

## 9. サーバー削除の確認

操作:

1. 保存済み契約一覧から契約を選ぶ
2. `サーバーから削除` を押す
3. 確認ダイアログでOKを押す
4. 契約一覧を更新する

チェック項目:

- 削除成功メッセージが出る
- 削除した契約が一覧から消える

## 10. セキュリティ確認

チェック項目:

- ログアウト後に契約入力フォームが見えない
- 管理者以外でログインした場合、契約一覧が表示されない
- `contract.html` が検索対象外になっている
- 共有端末では作業後にログアウトしている
- 不要な契約履歴は削除している

## 11. よくあるトラブル

ログインできない:

- Netlify Identityが有効か確認
- 招待メールの承認が完了しているか確認
- メールアドレスが `ADMIN_EMAILS` と一致しているか確認
- adminロールが付いているか確認

契約一覧が表示されない:

- Netlify Functionsが有効な環境で開いているか確認
- GitHub PagesではなくNetlify URLで開いているか確認
- ブラウザの開発者ツールで `/api/contracts` が401または403になっていないか確認

保存できない:

- 管理者ログイン済みか確認
- Netlify Blobsが使える公開環境か確認
- 署名画像が極端に大きくないか確認

GitHub Pagesで動かない:

- 正常です。GitHub Pagesではログイン・保存APIは動きません。
- 実運用テストはNetlifyで行ってください。
